# 依存関係の保証範囲

> 本ドキュメントは、AlchemyEngine が依存する外部パッケージ・ランタイムについて、**何を保証し、何を保証しないか**を明記する。

---

## 1. ランタイム・言語環境


| 依存                      | バージョン     | 保証するもの                                      | 保証しないもの                     |
| ----------------------- | --------- | ------------------------------------------- | --------------------------- |
| **Elixir**              | 1.19      | 言語仕様・BEAM VM の動作。CI で使用するバージョンでのコンパイル・テスト成功 | 1.19 以外のバージョンでの動作。将来の破壊的変更  |
| **OTP (Erlang/Elixir)** | 28        | NIF ロード・プロセスモデル・メモリ管理の動作                    | OTP 27 以前、OTP 29 以降での動作     |
| **Rust**                | stable    | コンパイル可能であること。`rustfmt` / `clippy` 通過        | 特定の nightly 機能。ツールチェーンの内部実装 |
| **Cargo**               | (Rust 同梱) | 依存解決・ビルドの再現性（Cargo.lock）                    | ビルド時間。キャッシュの挙動              |


---

## 2. Elixir 依存パッケージ

### 直接依存（`mix.exs` / 各 app の `mix.exs`）


| パッケージ                 | 用途                   | 保証するもの                          | 保証しないもの                           |
| --------------------- | -------------------- | ------------------------------- | --------------------------------- |
| **credo**             | 静的解析（dev/test）       | ルールに従ったコードスタイル・一貫性の指摘           | ロジックの正確性。セキュリティ。実行時エラー            |
| **rustler**           | Elixir–Rust NIF ブリッジ | NIF のロード・呼び出しインターフェース。BEAM との連携 | NIF 内の Rust 実装の正しさ。ブロッキング NIF の長さ |
| **telemetry**         | メトリクス・イベント           | イベント発火・購読の仕組み                   | イベントの意味・解釈。運用監視の設計                |
| **telemetry_metrics** | メトリクス集約              | メトリクスフォーマット・収集の仕組み              | 特定の監視システムとの互換性                    |
| **jason**             | JSON エンコード/デコード      | JSON 仕様準拠のパース・シリアライズ            | 任意の JSON スキーマの妥当性                 |
| **phoenix**           | WebSocket・HTTP サーバ   | WebSocket 接続・PubSub の仕組み        | 大規模同時接続時の性能。ネットワーク障害時の挙動          |
| **phoenix_pubsub**    | Pub/Sub              | メッセージ配信の基本的な動作                  | 厳密な順序保証。遅延保証                      |
| **plug_cowboy**       | HTTP サーバ (Cowboy)    | HTTP リクエスト/レスポンスの処理             | DDoS 耐性。特定プロトコルの完全対応              |
| **mox**               | テスト用モック (test)       | 呼び出しの検証・スタブの設定                  | 本番コードの正当性                         |


### 推移的依存（代表例）


| パッケージ                         | 保証するもの                   | 保証しないもの                   |
| ----------------------------- | ------------------------ | ------------------------- |
| **cowboy / ranch**            | HTTP 接続の受付・維持            | 極端な負荷下の安定性                |
| **plug**                      | リクエストライフサイクル・ミドルウェアチェーン  | カスタムアダプタの互換性              |
| **req** (rustler 経由)          | NIF ビルド時の Rust ツールチェーン取得 | ネットワーク障害時の再試行戦略           |
| **websock / websock_adapter** | WebSocket プロトコル実装        | すべての WebSocket 拡張・サブプロトコル |

## 3. Rust 依存クレート

### 共通・基盤


| クレート           | 用途          | 保証するもの                 | 保証しないもの                |
| -------------- | ----------- | ---------------------- | ---------------------- |
| **rustler**    | NIF バインディング | BEAM との型マッピング・エンコーディング | NIF 内でのパニック・長時間ブロックの防止 |
| **log**        | ログ出力        | ログマクロの呼び出し・メッセージ送信     | ログレベルの解釈・フォーマット        |
| **env_logger** | ログ初期化       | 環境変数によるログ設定            | 本番運用に適したログ設計           |
| **rayon**      | 並列処理        | データ並列の実行               | 特定スレッド数・スケジューリング       |
| **serde**      | シリアライズ      | 定義済み型のバイナリ/JSON 変換     | 任意スキーマの互換性             |
| **bincode**    | バイナリシリアライズ  | 定義済みプロトコルの一貫性          | 他言語・他バージョンとの相互運用       |
| **rustc-hash** | 高速ハッシュ      | 衝突確率・性能特性の一般的な挙動       | 暗号学的安全性                |


### 描画・ウィンドウ


| クレート                              | 用途            | 保証するもの                           | 保証しないもの                  |
| --------------------------------- | ------------- | -------------------------------- | ------------------------ |
| **wgpu**                          | GPU 描画 API    | Vulkan/Metal/DX12 等への抽象化。シェーダー実行 | 特定 GPU・ドライバでの動作。性能。機能の網羅 |
| **winit**                         | ウィンドウ・イベントループ | ウィンドウ作成・入力イベントの受付                | 全 OS・全デバイスでの同等動作         |
| **egui / egui-wgpu / egui-winit** | 即座モード UI      | 基本的な UI 描画・入力                    | 複雑なレイアウト・アクセシビリティ        |
| **image**                         | 画像読み込み        | PNG 等フォーマットのデコード                 | すべての画像フォーマット・破損ファイル      |
| **bytemuck**                      | バイト列キャスト      | 定義済み型の安全なキャスト                    | 任意メモリレイアウトの安全性           |
| **pollster**                      | 非同期ブロック       | `Future` の同期的実行                  | 本番の非同期設計                 |


### オーディオ


| クレート      | 用途      | 保証するもの              | 保証しないもの                   |
| --------- | ------- | ------------------- | ------------------------- |
| **rodio** | オーディオ再生 | WAV/Vorbis のデコード・再生 | 低遅延。空間オーディオ。すべてのオーディオデバイス |


> **補足**: `audio` は `cpal`（rodio の下位依存）経由で ALSA 等を使用。Linux CI では `libasound2-dev` をインストールしている。

### オプション依存


| クレート       | 用途                      | 保証するもの              | 保証しないもの            |
| ---------- | ----------------------- | ------------------- | ------------------ |
| **openxr** | VR/XR 入力 (input_openxr) | OpenXR ランタイム経由の入力取得 | 全ヘッドセット・全ランタイムでの動作 |

## 4. システム依存


| 依存                             | 用途           | 保証するもの                | 保証しないもの                  |
| ------------------------------ | ------------ | --------------------- | ------------------------ |
| **libasound2-dev** (Linux)     | オーディオ (ALSA) | NIF ビルド・オーディオ出力       | Windows/macOS の別実装との同等性  |
| **Vulkan / Metal / DX12 ドライバ** | wgpu         | 対応ドライバがある環境での描画       | headless 環境・ソフトウェアレンダリング |
| **OpenXR ランタイム**               | VR 対応ビルド     | ランタイムがインストールされた環境での入力 | 全 VR デバイス                |


---

## 5. まとめ

- **保証するもの**: 各依存の公式ドキュメント・API 契約に従った動作。CI で検証している環境でのビルド・テスト成功。
- **保証しないもの**: 依存の内部実装。将来の破壊的変更。未検証の環境・プラットフォーム。パフォーマンスの絶対値。セキュリティの網羅的保証。
