# 新コンテンツ: 3D Bullet Hell（弾避けシューター）

> 作成日: 2026-03-02

---

## 目的

### エンジン観点

`SimpleBox3D` は「3Dパイプラインが動く」ことを確認するだけの検証用コンテンツだった。
本コンテンツでは **既存の2D ECS・NIF インターフェースを3Dゲームに転用できるか** を検証し、
「3D化でどこが変わるか」という課題を浮き彫りにすることを目的とする。

具体的に明らかにしたい課題：

| 課題カテゴリ | 確認したい問い |
|:---|:---|
| NIF インターフェース | `spawn_projectile` / `spawn_enemies` は Z 座標なしで設計されているが、3D コンテンツに転用できるか？ |
| 物理 ECS | 2D の `EnemyWorld` / `BulletWorld`（SoA）を Z 軸付きに拡張するコストはどの程度か？ |
| 描画混在 | 2D スプライトと 3D メッシュを同一フレームで混在させる必要はあるか？ |
| カメラ制御 | 現在の固定俯瞰カメラで「弾を避ける」ゲームプレイが成立するか？ |
| Elixir SSoT | 3D 座標の権威をどこに置くか（Elixir vs Rust）？ |

### コンテンツ観点

シンプルなルールで長く遊べる「弾幕避け」を3Dで実現する。
プレイヤーは動き続けることで生き残り、時間経過とともに弾が増えていく。

---

## 何を作るか

### ゲーム概要

| 項目 | 内容 |
|:---|:---|
| ジャンル | 3D 弾幕避け（Bullet Hell） |
| 視点 | 固定俯瞰カメラ（斜め上から見下ろし） |
| 操作 | WASD でプレイヤーを XZ 平面上で移動 |
| 目標 | 弾に当たらずできるだけ長く生き残る |
| 終了条件 | 弾に 3 回当たる（HP = 3） |

### ゲームプレイ

```
[プレイヤー]
  - 青いボックス（SimpleBox3D と同形）
  - XZ 平面上を WASD で移動
  - HP = 3（弾に当たるたびに -1）

[敵]
  - 赤いボックスが複数体、フィールド外周に出現
  - プレイヤーに向かって直進する
  - プレイヤーに接触するとダメージ

[弾]
  - 敵が定期的にプレイヤー方向へ弾を発射
  - 弾はボックスより小さい黄色いボックス
  - フィールド端に到達すると消滅

[フィールド]
  - グリッド地面（20×20 ユニット）
  - スカイボックス（空色グラデーション）
  - フィールド端に見えない壁（プレイヤーは出られない）
```

### 難易度スケーリング

| 経過時間 | 変化 |
|:---|:---|
| 0 秒〜 | 敵 2 体、発射間隔 3 秒 |
| 30 秒〜 | 敵 4 体、発射間隔 2 秒 |
| 60 秒〜 | 敵 6 体、発射間隔 1.5 秒 |
| 90 秒〜 | 敵 8 体、発射間隔 1 秒 |

### シーン構成

```
Playing  ─── ダメージ 3 回 ──→  GameOver
  ↑                                 │
  └──────────── リトライ ────────────┘
```

### 描画構成

| 描画要素 | DrawCommand | 備考 |
|:---|:---|:---|
| 空 | `Skybox` | SimpleBox3D と同じ |
| 地面 | `GridPlane` | SimpleBox3D と同じ |
| プレイヤー | `Box3D`（青） | 1.0×1.0×1.0 |
| 敵 | `Box3D`（赤） | 1.0×1.0×1.0 |
| 弾 | `Box3D`（黄） | 0.3×0.3×0.3 |
| HP 表示 | egui HUD | 残り HP をアイコンで表示 |
| 生存時間 | egui HUD | 経過秒数 |

### コンポーネント構成

```
GameContent.BulletHell3D
  ├── SpawnComponent     # ワールド初期化・エンティティパラメータ注入
  ├── EnemyComponent     # 敵スポーン・AI（Elixir 側で速度注入）
  ├── BulletComponent    # 弾発射タイミング管理
  ├── DamageComponent    # HP 管理・ゲームオーバー判定
  └── RenderComponent    # DrawCommand 組み立て・push_render_frame
```

## 期待する発見

このコンテンツを作ることで以下の課題が具体的に現れると予想する：

1. **`spawn_projectile` NIF の Z 座標問題**
   現在のシグネチャ `spawn_projectile(x, y, vx, vy, damage, lifetime, kind)` に Z 軸がない。
   3D 弾を扱うには NIF 拡張か、Elixir 側で Z を管理するかの判断が必要になる。

2. **`EnemyWorld` の Z 軸不在**
   `EnemyWorld` の SoA 構造に `positions_z` / `velocities_z` が存在しない。
   3D 移動を Rust 側でやるか Elixir 側でやるかの設計判断が迫られる。

3. **描画の Z ソート問題**
   3D 空間で複数の `Box3D` が重なる場合、深度テストは wgpu 側で処理されるが、
   半透明要素（ダメージフラッシュ等）を追加したい場合に描画順の問題が生じる可能性がある。

4. **HUD の 3D 対応**
   現在の HUD は 2D ゲーム用のフィールド（HP バー・EXP バー・武器スロット）を持つ。
   3D ゲームに不要なフィールドをどう扱うか（ダミー値 vs HUD 型の分離）が問われる。
