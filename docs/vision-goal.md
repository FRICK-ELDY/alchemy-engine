# AlchemyEngine — 最終ゴール：分散型VR空間

> このドキュメントは「AlchemyEngine が最終的に何を目指すか」を記述する。
> まだ未熟な思想であり、詰め切れていない問いも多い。
> 実装の進捗に関わらず、ビジョンを育てるために随時更新する。

---

## 一言で言うと

> **個人が自分のドメインで運営する3D空間が、世界中でフェデレーション（連合）し、
> 誰でも自分の空間を持ち、コンテンツを作り、他の空間と繋がれる——
> 分散型VR SNS。**

---

## 既存の類似概念との対応

このビジョンは、既存の分散型 SNS の思想をVR空間に適用したものだ。

| AlchemyEngine のビジョン | 既存の類似概念 |
|:---|:---|
| `alchemy.frick-eldy.com`（個人インスタンス） | Mastodon / Misskey のインスタンス |
| インスタンス間の連携 | ActivityPub のフェデレーション |
| コンテンツの Sensitive/NSFW 分類 | Mastodon の `sensitive` フラグ + インスタンスポリシー |
| 個人ドメインで自己ホスト | セルフホスト型分散 SNS |
| 空間上のコンテンツ | ActivityPub の `Activity` / `Object` |
| Hub（コンテンツ一覧） | Mastodon のタイムライン / Misskey のノート |

ActivityPub は「テキスト投稿・フォロー・いいね」を分散化した。
AlchemyEngine は「3D空間・コンテンツ・リアルタイム体験」を分散化する。

---

## インスタンスの概念

個人または組織が、自分のドメインで AlchemyEngine を運用する。

```
alchemy.frick-eldy.com     ← frick-eldy 個人のインスタンス
alchemy.some-studio.com    ← あるスタジオのインスタンス
alchemy.enterprise.co.jp   ← 企業のインスタンス
```

各インスタンスは：

- 独自の3D空間・コンテンツを持つ
- 独自のポリシー（何を公開するか・何を受け入れるか）を持つ
- `apps/game_server` が動いており、外部からアクセスできる

---

## フェデレーション（インスタンス間連携）

インスタンス同士は連携できる。

```
alchemy.frick-eldy.com  ←→  alchemy.some-studio.com
alchemy.frick-eldy.com  ←→  alchemy.enterprise.co.jp
```

連携すると：

- 相手インスタンスのコンテンツが観測できる
- 相手インスタンスの空間に参加できる（クロスインスタンス入室）
- ユーザーは `@user@alchemy.frick-eldy.com` のような形で識別される

コンテンツが増え続けるほど、フェデレーションの価値が高まる。

## コンテンツポリシーとフィルタリング

### 設計の基本思想：タグとステータスの2層構造

コンテンツの分類は **「タグ（含まれる表現の種類）」** と **「ステータス（総合的な公開レベル）」** の2層で管理する。

- **タグ**：コンテンツに含まれる表現の種類。複数付く。コンテンツ作者が申告する。
- **ステータス**：タグの組み合わせから決まる、コンテンツ全体の公開レベル。

インスタンスポリシーは **ステータス単位** で設定する。

---

### コンテンツタグ（表現の種類）

コンテンツに含まれる表現にタグを付ける。タグは複数付けられる。

| タグ | 例 |
|:---|:---|
| `Religion` | 宗教的シンボル・儀式・信仰の描写 |
| `Violence` | 戦闘・暴力行為 |
| `Blood` | 出血表現 |
| `Gore` | 内臓・損壊等の過激な流血表現 |
| `Horror` | ホラー・グロテスク |
| `BunnyOutfit` | バニー服・水着等の露出度の高い衣装 |
| `Underwear` | 下着姿 |
| `HairNude` | ヘアヌード（性器非露出の全裸） |
| `Intercourse` | 性交・交尾の描写 |
| *(他、随時追加)* | |

---

### コンテンツステータス（公開レベル）

タグの組み合わせからステータスが決まる。ステータスは1つだけ持つ。

| ステータス | 意味 | 対応するタグの例 |
|:---|:---|:---|
| `General` | 誰でも閲覧可能 | タグなし |
| `Sensitive` | 一部の人が不快に感じる可能性がある | `Religion`, `Violence` |
| `Mature` | 成人向け（暴力・ホラー等） | `Blood`, `Gore`, `Horror` |
| `Sexual` | 性的表現を含む | `BunnyOutfit`, `Underwear`, `HairNude` |
| `Explicit` | 露骨な性的表現を含む | `Intercourse` |

ステータスの決定ルール：

- タグが `Religion` → ステータスは最低でも `Sensitive`
- タグが `Blood` → ステータスは最低でも `Mature`
- タグが `BunnyOutfit` / `Underwear` / `HairNude` → ステータスは最低でも `Sexual`
- タグが `Intercourse` → ステータスは `Explicit` 固定

**`Sexual` 以上のステータス判定はドメイン管理者が最終決定する。**
コンテンツ作者が `BunnyOutfit` タグを付けても、管理者が `Mature` に下げることも、
`Explicit` に引き上げることもできる。性的表現の基準は文化・地域によって異なるため、
機械的なルールではなく人間の判断を介在させる設計にする。

---

### インスタンスポリシー

各インスタンスは「どのステータスまで受け入れるか」のポリシーを持つ：

```
enterprise.co.jp のポリシー:
  許可するステータス上限: General
  （Sensitive 以上のコンテンツは連携時に除外される）

some-studio.com のポリシー:
  許可するステータス上限: Mature

frick-eldy.com のポリシー:
  許可するステータス上限: Sexual
```

---

### フィルタリングの仕組み

連携時は **受信側インスタンスのポリシー** に従ってフィルタリングされる。

```
enterprise.co.jp から frick-eldy.com を観測する場合：
  → frick-eldy.com のコンテンツのうち、
    ステータスが General のものだけが見える

some-studio.com から frick-eldy.com を観測する場合：
  → ステータスが General / Sensitive / Mature のものが見える
```

これにより：

- 企業インスタンスは、クリーンなコンテンツだけを取り込める
- 個人インスタンスは、好みに合わせて自由に設定できる
- コンテンツ作者は、タグを正直に付けることで適切な場所に届く
- `Sexual` 以上の判定を管理者に委ねることで、文化差・地域差に対応できる

## ユーザーのアイデンティティ

ユーザーは自分のホームインスタンスに属する。

```
@frick-eldy@alchemy.frick-eldy.com
```

- ホームインスタンスのポリシーに従う
- 別インスタンスの空間に入っても、アイデンティティは変わらない
- ブロック・ミュートはインスタンス単位でも、ユーザー単位でも設定できる（未決）

## メモ・妄想（VR 完成後の将来構想）

### 眼鏡なしで見えるレンダリング補正

VR モードが完成したら、以下のような「眼鏡を外したまま快適に見える」ためのレンダリング補正を実現したい。

- **片目ずつの IPD 設定**  
  瞳孔間距離（Inter-Pupillary Distance）を左右で個別に設定可能にする。左右非対称の顔の形状に対応。

- **片目ずつのピント（焦点）設定**  
  近視・遠視・乱視など、左右別々の屈折異常をレンダリングで補正する。  
  各目ごとに仮想的なピント距離（または度数）を指定し、シェーダーやレンダリングパイプラインで補正をかける。

- **実装方針**  
  これらは物理的な光学調整ではなく、**レンダリング側の仮想補正**として実現する。  
  VR ヘッドセットのディスプレイ解像度・レンズ仕様を前提に、ユーザーが「自分の目に合った見え方」になるようパラメータを調整できるようにする。

> 現時点では妄想レベルの構想。VR 基盤が整った後に検討する。
