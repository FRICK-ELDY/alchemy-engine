# ラボ運用設計（たたき台）

この文書は **パブリックリポジトリ用の参考テンプレート** です。実際の IP・パスワード・鍵・ドメインは **ローカルや秘密管理** に置き、ここには **プレースホルダ** のみを書いてください。

ネットワークの物理・論理の全体像は [lab-environment.md](./lab-environment.md) を参照してください。

---

## 1. VLAN / サブネット（例）

実番号は環境に合わせて差し替えます。ルーティングと DHCP は **RTX1300 側** に寄せる想定です。

| VLAN ID（例） | 名前           | 用途                         | サブネット（例）   | ゲートウェイ（例） |
|----------------|----------------|------------------------------|--------------------|--------------------|
| 10             | trusted        | メインPC、管理、NAS 管理系   | 192.168.10.0/24    | 192.168.10.1       |
| 20             | lab            | Win / Mac / Linux テスト機   | 192.168.20.0/24    | 192.168.20.1       |
| 30             | iot_guest      | WiFi ゲスト・IoT（将来）     | 192.168.30.0/24    | 192.168.30.1       |
| （任意）       | mgmt           | 機器管理 VLAN（推奨時のみ）  | TBD                | TBD                |

- **CRS305**: RTX との間はトランク（複数 VLAN タグ）を想定する場合、RouterOS 側で **VLAN テーブルとポート VLAN** を定義する。
- **SWX2210-8G**: テスト機向けポートを **lab VLAN** に割り当て、必要なら **アップリンクをタグ付き** で RTX に返す（機種の VLAN 仕様に合わせて調整）。

## 2. ゾーン間ファイアウォール方針（たたき台）

「許可するものだけ列挙、それ以外は拒否」を前提に、**RTX のフィルタ** で実装するイメージです。具体のコマンドは機種マニュアルに従ってください。

| 送信元ゾーン | 宛先ゾーン   | 許可（例）                                      | メモ |
|--------------|--------------|-------------------------------------------------|------|
| trusted      | Internet     | 必要なプロトコルのみ                            | 運用方針に合わせる |
| lab          | NAS（アセット） | TCP `445` / `2049` / `443` 等、**実際に使う転送のみ** | SMB/NFS/HTTPS は用途に合わせて絞る |
| lab          | NAS 管理     | **原則拒否**（DSM、SSH、Docker API 等）         | メインPCからのみ管理 |
| lab          | trusted      | **原則拒否**（管理踏み台が必要なら例外を明示）   | |
| lab          | syslog 集約  | UDP/TCP **514**（または集約側のポート）         | 集約先を1つに固定 |
| iot_guest    | 他           | **Internet のみ** または **完全隔離**           | 要件次第 |

Pi（低速シミュレーション）は **lab と同一 VLAN のまま** でもよいが、**横展開を抑える** なら lab 内でさらに **ホスト単位のルール** や **別サブネット** を検討する。

## 3. ログパイプライン（たたき台）

| 役割           | 配置（例）              | 内容 |
|----------------|-------------------------|------|
| 一次受け       | RTX1300                 | 短いバッファ・ローテーション。長期保存の主役にしない。 |
| 集約           | NAS 上コンテナ          | syslog-ng / Vector / Promtail 等から **1 系統** に集約。 |
| 可視化         | Grafana（+ Loki 等）    | ダッシュボード・検索。保持期間はディスクに合わせて **7日 / 30日** 等を定義。 |
| アーカイブ     | 任意（R2 等）           | 圧縮・個人情報・機密が混ざらないかを確認してから。 |

**保持期間（例）**

- ローカルホット: 7〜30 日
- 調査用ミディアム: 必要なら 90 日（容量と相談）
- 長期: マスキング済みのみオフサイト

## 4. NAS 上サービスと信頼境界（たたき台）

| サービス        | 想定利用者     | 公開範囲（例）     | メモ |
|-----------------|----------------|--------------------|------|
| アセット共有    | lab + trusted  | 必要ポートのみ     | 管理 UI とは分離 |
| Gitea           | trusted 主     | trusted のみ推奨   | 認証・バックアップ方針を別紙 |
| Container Registry | CI / ビルド | lab の限定 IP のみ | pull のみ・認証トークン期限 |
| Grafana         | 管理者         | trusted のみ       | ログに秘密が出ないようアプリ側マスキング |

**バックアップ（概念）**

- 設定・リポジトリ・レジストリの **リストを固定** し、復旧手順を1ページにまとめる。
- オフサイト（例: R2）は **暗号化・ライフサイクル** を別管理。
