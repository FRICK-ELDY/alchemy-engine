# ランチャー実装計画（実施用）

> 作成日: 2026-03-08  
> 目的: 動くことを確認しながら、フェーズごとにランチャーを組み立てる。  
> 参照: [launcher-design.md](./launcher-design.md)

## 原則

- **各フェーズで「動く」ことを確認してから次へ進む**
- **フェーズごとにビルド・実行・動作確認を行う**
- **問題が出たらそのフェーズ内で解決してから次へ**

---

## Phase 0: ランチャーの起動と終了確認

**目標**: ランチャー exe が起動し、トレイに表示され、Quit で終了できることを確認する。

### 0.1 やること

| # | 内容 | 完了条件 |
|:---|:---|:---|
| 0.1.1 | 最小構成の launcher クレート | `cargo build -p launcher` が成功する |
| 0.1.2 | トレイアイコン表示 | 起動するとタスクトレイにアイコンが表示される |
| 0.1.3 | メニューに「Quit」のみ | 右クリックでメニューが開き、「Quit」を選べる |
| 0.1.4 | Quit で終了 | 「Quit」選択でランチャーが終了する |

### 0.2 含めないもの（Phase 0 では実装しない）

- zenohd Run / Quit
- HL-Server Run / Quit
- Client Run
- Check for Update / acknowledgements
- 状態に応じたアイコン切り替え
- プロジェクトルートの解決（とりあえず固定パス or カレントでよい）

### 0.3 想定ファイル構成

```
native/launcher/
  src/
    main.rs    # トレイ + Quit のみ
  Cargo.toml
```

### 0.4 動作確認手順

1. `bin/launcher.bat` または `cargo run -p launcher` で起動
2. トレイにアイコンが出ることを確認
3. 右クリック → メニューに「Quit」があることを確認
4. 「Quit」クリック → ランチャーが終了することを確認

## Phase 2: zenohd のポート確認と UX 改善

**前提**: Phase 1 が完了していること。

**目標**: ポート 7447 で待ち受けを確認し、成功/失敗を正しく扱う。

### 2.1 やること

| # | 内容 | 完了条件 |
|:---|:---|:---|
| 2.1.1 | ポート 7447 の待ち受け確認 | 起動後にポートが応答することを確認 |
| 2.1.2 | 起動成功/失敗の判定 | ポート応答を待って成功/失敗を判定 |
| 2.1.3 | タイムアウト時のメッセージ | 失敗時は分かりやすいメッセージを表示 |

### 2.2 課題への対応

- 「zenohd は起動しているのに 10 秒以内に起動しませんでした」が出る場合:
  - 初回待機時間・ポーリング間隔・タイムアウトの見直し
  - zenohd の実際の起動完了までの時間を計測して設定を調整

## 実施時の注意

1. **Phase 0 から順に実施する**。Phase 0 が動くまで Phase 1 に進まない。
2. **既存の launcher 実装をどう扱うか**:
   - 変更をなかったことにする場合: `native/launcher/` を Phase 0 の最小構成に差し替える
   - 残す場合: Phase 0 の要件を満たす最小形に絞り込み、不要なコードを削除する
3. **各 Phase 終了時にビルド・実行・動作確認を行う。**
